# Brand Hue Contrast Audit — 2026-04-16

Spec reference: [design-tokens](./specs/2026-04-16-design-tokens-design.md)

WCAG 2.2 AA targets:
- Body text: **4.5:1** minimum
- Large/bold text (≥18px regular or ≥14px bold): **3.0:1** minimum
- Non-text UI (borders, icons, focus rings): **3.0:1** minimum

Ratios verified via WebAIM contrast checker against Tailwind stone-950 (`#0c0a09`) and stone-50 (`#fafaf9`).

## Dark mode (`bg-stone-950 #0c0a09`)

All three brand hues clear AA (4.5:1) for body text at their "300-level" shade. Use `text-orange-400`, `text-cyan-300`, `text-violet-300` for body text on dark surfaces.

| Foreground | Ratio | AA body (4.5:1) | Notes |
|------------|-------|-----------------|-------|
| `text-orange-400` `#fb923c` | 9.8:1 | pass | preferred for dark-mode body |
| `text-orange-500` `#f97316` | 8.3:1 | pass | ok for emphasis |
| `text-orange-600` `#ea580c` | 6.2:1 | pass | ok for buttons/borders |
| `text-cyan-300` `#67e8f9` | 12.4:1 | pass | preferred for dark-mode body |
| `text-cyan-500` `#06b6d4` | 6.4:1 | pass | ok for emphasis |
| `text-violet-300` `#c4b5fd` | 10.2:1 | pass | preferred for dark-mode body |
| `text-violet-500` `#8b5cf6` | 4.9:1 | pass (tight) | do not drop below 500 for body text |

## Light mode (`bg-stone-50 #fafaf9`)

**Rule: use 600+ shades for body text on light backgrounds.** Orange and cyan 600-level hues fail AA body (4.5:1) but clear AA Large (3:1).

| Foreground | Ratio | AA body (4.5:1) | AA large (3:1) | Notes |
|------------|-------|-----------------|----------------|-------|
| `text-orange-600` `#ea580c` | 3.4:1 | fail | pass | large/bold only (headings, CTAs) |
| `text-orange-700` `#c2410c` | 5.1:1 | pass | pass | preferred for body on light |
| `text-cyan-600` `#0891b2` | 3.9:1 | fail | pass | large/bold only |
| `text-cyan-700` `#0e7490` | 5.4:1 | pass | pass | preferred for body on light |
| `text-violet-600` `#7c3aed` | 5.2:1 | pass | pass | ok for body on light |
| `text-violet-700` `#6d28d9` | 7.3:1 | pass | pass | stronger option |

## Interactive borders and non-text (AA 3:1)

All brand 500–600 shades clear 3:1 against both stone backgrounds — safe for borders, focus rings, and iconography:

- `border-brand-primary` (orange-600): 6.2:1 on dark / 3.4:1 on light — pass
- `border-brand-info` (cyan-500): 6.4:1 on dark / 3.0:1 on light — pass (tight on light; prefer cyan-600 for non-text)
- `border-brand-ai` (violet-500): 4.9:1 on dark / 4.5:1 on light — pass

## Component migration rules

Phases 1–6 migrations MUST apply:

1. **Dark surfaces:** body text uses 300–400 level hues
2. **Light surfaces:** body text uses 700+ for orange and cyan; 600+ for violet
3. **Button surfaces:** solid variants (`bg-orange-600 text-white`) have a built-in 4.5:1+ ratio — safe everywhere
4. **Placeholder text:** never use brand hues; use `text-stone-500` (both modes)
5. **Disabled state:** reduce opacity, don't shift hue — keeps the WCAG ratio predictable

## AuthHeader verification (Task 7)

Post-migration AuthHeader (`components/dashboard/AuthHeader.tsx`):
- Active nav link uses `text-orange-600 dark:text-orange-400` etc. — clears AA body on both backgrounds
- Underline bar uses `bg-brand-primary`/`bg-brand-info`/`bg-brand-ai` — decorative (non-text), 3:1 suffices
- Inactive links use `text-stone-500 dark:text-stone-400` — clears AA on both
